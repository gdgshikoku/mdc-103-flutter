マテリアル コンポーネント (MDC) は、開発者がマテリアル デザインを実装するのに役立ちます。 Google のエンジニアと UX デザイナーのチームによって作成された MDC は、多数の美しく機能的な UI コンポーネントを備えており、Android、iOS、ウェブ、および Flutter.material.io/develop で利用できます。

MDC を使用して、アプリの独自のスタイルをこれまで以上にカスタマイズできるようになりました。 Material Design の最近の拡張により、デザイナーや開発者は製品のブランドをより柔軟に表現できるようになりました。

コードラボ MDC-101 と MDC-102 では、Material Components (MDC) を使用して、Shrine というアプリの基本を作成しました。Shrine は、衣料品や家庭用品を販売する e コマース アプリです。 このアプリには、ログイン画面から始まり、製品を表示するホーム画面にユーザーを誘導するユーザー フローが含まれています。

colors.dart を作成する

colors.dart という新しいファイルを lib フォルダに作成します。 マテリアル コンポーネントをインポートし、const Color 値を追加します。

import 'package:flutter/material.dart';

const kShrinePink50 = Color(0xFFFEEAE6);
const kShrinePink100 = Color(0xFFFEDBD0);
const kShrinePink300 = Color(0xFFFBB8AC);
const kShrinePink400 = Color(0xFFEAA4A4);

const kShrineBrown900 = Color(0xFF442B2D);

const kShrineErrorRed = Color(0xFFC5032B);

const kShrineSurfaceWhite = Color(0xFFFFFBFA);
const kShrineBackgroundWhite = Colors.white;

カスタム カラー パレット

このカラー テーマは、カスタム カラーを使用してデザイナーによって作成されました (下の画像を参照)。 これには、Shrine のブランドから選択された色が含まれており、Material Theme Editor に適用され、より充実したパレットを作成するために拡張されています。 (これらの色は、2014 のマテリアル カラー パレットからのものではありません。)

Material Theme Editor は、各色のラベル 50、100、200、.... 900 を含む、数値でラベル付けされた色合いにそれらを整理しました。 シュラインは、ピンク色の見本の 50、100、および 300 と、茶色の見本の色 900 のみを使用します。

ウィジェットの各色付きパラメーターは、これらのスキームの色にマップされます。 たとえば、アクティブに入力を受け取っているときのテキスト フィールドの装飾の色は、テーマのプライマリ カラーにする必要があります。 その色にアクセスできない (背景に対して見やすい) 場合は、代わりに PrimaryVariant を使用します。

色

kShrineBackgroundWhite の値は、Colors クラスから取得されます。 このクラスには、白などの一般的な色の値があります。 また、2014 年のカラー パレットが MaterialColor クラスとして含まれています。

素材色

「material/colors.dart」にある MaterialColor クラス (ColorSwatch のサブクラス) は、赤、緑、薄緑、またはライムの 14 階調など、原色の 14 以下のバリエーションのグループです。 これらのバリエーションは、塗料店で見つけたグラデーション カラー チップに似ています。

これらのバリエーションは 2014 マテリアル ガイドライン用に作成されたもので、現在のガイドライン (カラー システムの記事) と MDC-Flutter の両方で引き続き利用できます。コードでそれらにアクセスするには、ベース カラーを呼び出してから、シェード (通常は 100 の値) を呼び出します。たとえば、Pink 400 は次のコマンドで取得されます: Colors.pink[400]。

これらのパレットをデザインやコードに使用しても問題ありません。ブランディングに固有の色が既にある場合は、パレット生成ツールまたはマテリアル テーマ エディターを使用して、独自の調和のとれたパレットを生成できます。

使用したい色ができたので、それらを UI に適用できます。これを行うには、ウィジェット階層の最上位にある MaterialApp インスタンスに適用する ThemeData ウィジェットの値を設定します。

ThemeData.light() をカスタマイズする

Flutter には、いくつかの組み込みテーマが含まれています。ライト テーマはその 1 つです。 ThemeData ウィジェットを最初から作成するのではなく、ライト テーマをコピーして値を変更し、アプリ用にカスタマイズします。

color.dart を app.dart にインポートしましょう。

import 'colors.dart';

次に、ShrineApp クラスのスコープ外の app.dart に次を追加します。

// TODO: Build a Shrine Theme (103)
final ThemeData _kShrineTheme = _buildShrineTheme();

ThemeData _buildShrineTheme() {
  final ThemeData base = ThemeData.light();
  return base.copyWith(
    colorScheme: base.colorScheme.copyWith(
      primary: kShrinePink100,
      onPrimary: kShrineBrown900,
      secondary: kShrineBrown900,
      error: kShrineErrorRed,
    ),
    // TODO: Add the text themes (103)
    // TODO: Add the icon themes (103)
    // TODO: Decorate the inputs (103)
  );
}

次に、theme: を設定します。ShrineApp の build() 関数 (MaterialApp ウィジェット内) の最後に新しいテーマを設定します。

  // TODO: Add a theme (103)
  theme: _kShrineTheme, // New code

ThemeData インスタンスのコピー

デフォルトのライト ThemeData で copyWith() を呼び出し、いくつかのカスタム プロパティ値を渡しました (copyWith() は、Flutter の多くのクラスとウィジェットで見られる一般的なメソッドです)。 このコマンドは、呼び出されたインスタンスと一致するウィジェットのインスタンスを返しますが、指定された値は置き換えられます。

ThemeData インスタンスをインスタンス化してから、そのプロパティを設定しないのはなぜですか? 我々は出来た！ アプリの構築を続ければ、それは理にかなっています。 ThemeData には多数のプロパティがあるため、時間を節約するために魅力的なテーマから始めて、コードラボで確認できる値を変更します。 後で別のテーマを試すときは、MDC-Flutter に含まれる別の ThemeData から始めます。

Flutter のドキュメントで ThemeData の詳細を確認してください。

テキストテーマをカスタマイズする

プロジェクトにフォントをインポートするには、それらを pubspec.yaml ファイルに追加する必要があります。

pubspec.yaml で、flutter: タグの直後に次を追加します。

  # TODO: Insert Fonts (103)

          weight: 500

これで、Rubik フォントにアクセスして使用できるようになりました。

pubspec ファイルのトラブルシューティング

上記の宣言をカット アンド ペーストすると、pub get の実行中にエラーが発生する場合があります。 エラーが発生した場合は、最初に先頭の空白を削除し、2 スペースのインデントを使用してスペースに置き換えます。 (前に 2 つのスペース

fonts:

、前に 4 つのスペース

family: Rubik

、 等々。）

Mapping values are not allowed here と表示された場合は、問題のある行のインデントとその上の行のインデントを確認してください。

login.dart で、Column() 内で次のように変更します。

Column(
  children: <Widget>[
    Image.asset('assets/diamond.png'),
    const SizedBox(height: 16.0),
    Text(
      'SHRINE',
      style: Theme.of(context).textTheme.headline5,
    ),
  ],
)

app.dart で、_buildShrineTheme() の後に次を追加します。

// TODO: Build a Shrine Text Theme (103)
TextTheme _buildShrineTextTheme(TextTheme base) {
  return base.copyWith(
    headline5: base.headline5!.copyWith(
      fontWeight: FontWeight.w500,
    ),
    headline6: base.headline6!.copyWith(
      fontSize: 18.0,
    ),
    caption: base.caption!.copyWith(
      fontWeight: FontWeight.w400,
      fontSize: 14.0,
    ),
    bodyText1: base.bodyText1!.copyWith(
      fontWeight: FontWeight.w500,
      fontSize: 16.0,
    ),
  ).apply(
    fontFamily: 'Rubik',
    displayColor: kShrineBrown900,
    bodyColor: kShrineBrown900,
  );
}

これは TextTheme を受け取り、見出し、タイトル、およびキャプションの外観を変更します。

このように fontFamily を適用すると、copyWith() で指定されたタイポグラフィ スケール値 (見出し、タイトル、キャプション) のみに変更が適用されます。

一部のフォントでは、カスタムの fontWeight を 100 単位で設定しています。w500 (500 ウェイト) はミディアムに対応し、w400 は通常に対応します。

新しいテキスト テーマを使用する

テキストテーマ

テキスト テーマは、アプリ内のすべてのテキストを一貫して読みやすくするための便利な方法です。 たとえば、テーマの原色の明るさに応じて、テキスト テーマのスタイルを黒または白にすることができます。 これにより、テキストが背景と適切なコントラストを持つようになり、常に読みやすくなります。

TextTheme の詳細については、Flutter のドキュメントをご覧ください。

エラーの後、次のテーマを _buildShrineTheme に追加します。

// TODO: Add the text themes (103)
textTheme: _buildShrineTextTheme(base.textTheme),
textSelectionTheme: const TextSelectionThemeData(
  selectionColor: kShrinePink100,
),

標高に関する注意事項:

  • すべてのマテリアル デザイン サーフェスとコンポーネントには標高値があります。
  • 1 つのサーフェスが終了する場所と別のサーフェスが開始する場所の間の分割は、サーフェス エッジによって区別されます。
  • サーフェス間の標高差は、背景を薄くしたり、背景を明るくしたり、影を付けたりして表現できます。
  • 通常、他のサーフェスの前にあるサーフェスには、より重要なコンテンツが含まれています。

Shrine は、八角形または長方形の要素を定義するクールな幾何学的なスタイルを持っています. ホーム画面のカードと、ログイン画面のテキスト フィールドとボタンに、その形状のスタイリングを実装しましょう。

TextButton には目に見える形がないのに、なぜ境界線の形を追加するのでしょうか? そのため、リップル アニメーションは、タッチすると同じ形状にバインドされます。

形状に関する注意事項:

  • 形は、ブランドの視覚的表現に貢献できます。
  • シェイプには、カーブとコーナーの角度、カーブとエッジの角度、およびコーナーの総数を調整できます。
  • コンポーネントの形状がその使いやすさを妨げてはなりません!

色はブランドを表現する強力な方法であり、色のわずかな変更がユーザー エクスペリエンスに大きな影響を与える可能性があります。 これをテストするために、ブランドの配色が少し異なる場合に神社がどのように見えるかを見てみましょう.

色を変更する

colors.dart で、次の色を追加します。

const kShrinePurple = Color(0xFF5D1049);

app.dart で、_buildShrineTheme() 関数を次のように変更します。

ThemeData _buildShrineTheme() {
  final ThemeData base = ThemeData.light();
  return base.copyWith(
    colorScheme: base.colorScheme.copyWith(
      primary: kShrinePurple,
      secondary: kShrinePurple,
      error: kShrineErrorRed,
    ),
    scaffoldBackgroundColor: kShrineSurfaceWhite,
    textSelectionTheme: const TextSelectionThemeData(
      selectionColor: kShrinePurple,
    ),
    inputDecorationTheme: const InputDecorationTheme(
      border: CutCornersBorder(),
      focusedBorder: CutCornersBorder(
        borderSide: BorderSide(
          width: 2.0,
          color: kShrinePurple,
        ),
      ),
      floatingLabelStyle: TextStyle(
        color: kShrinePurple,
      ),
    ),
  );
}
